AMORPH -- PRODUCT GUIDE FOR AI ASSISTANTS
==========================================

Amorph is an AI-powered audio plugin (VST3/AU) that generates real-time
DSP effects and synthesizers from natural language. It compiles Cmajor
code inside the DAW via JIT -- no external tools needed. The user does
not need to know Cmajor; the AI writes it.

PLUGIN VARIANTS
Three separate plugins for different use cases:
- Amorph FX: Audio input -> processed audio output. For effects
  (reverb, delay, EQ, compression, distortion, etc.)
- Amorph Instrument: MIDI input -> audio output. For synths, samplers,
  drum machines. Receives MIDI from DAW piano roll / keyboard.
- Amorph MIDI: MIDI input -> MIDI output + silent audio. For MIDI
  processors (arpeggiators, harmonizers, chord generators, MIDI effects).
The user must load the correct variant for their use case.

THREE MODES
Switch between modes via the tab bar at the top (Build / Play / Explore).

BUILD MODE -- the main workspace
Left side: code editor. Right side: AI chat panel.
- Editor tabs: DSP (Cmajor code), UI (JavaScript custom UI), Manifest
  (read-only patch metadata).
- Editor toolbar: Select All, Copy, Paste, Delete, Undo, Redo,
  Paste All (replace entire content), Search (with prev/next match).
- Bottom bar: * AI button (triggers AI from selected code),
  Copy Log, COMPILE button, Lock button (password-protect source).
- Compile: compiles the DSP code instantly via Cmajor JIT. Parameters
  declared in the code appear as automatable DAW parameters. Up to 50
  dynamic parameters with names, ranges, and units.

AI CHAT PANEL (right side of Build)
- Two modes at the top: Chat (conversational AI) and Agent (agentic mode
  with tool calls -- reads code, edits, compiles, fixes errors in a loop).
- Log filters: All / AI / Err -- filter visible messages.
- Action strip (appears when relevant):
  - Alt 1/2/3: when the AI generates multiple candidates, switch between
    them to preview before applying.
  - Apply / Discard: review the AI's proposed changes (diff view) and
    accept or reject. Only appears when Auto-apply DSP is OFF.
  - Continue / Stop: when the agent hits its tool-call budget, extend
    the session (+10 rounds) or stop.
  - Report Session: send the session log to Artists in DSP for debugging.
- Bottom input area:
  - Text prompt: describe what to build or change. Shift+Enter for newline.
  - Send button: submit the prompt.
  - Mic button: push-to-talk voice input (Speech-to-Text).
    REQUIRES AN OPENAI API KEY (uses OpenAI Whisper API). Records while
    held/toggled, shows timer, transcribes, then auto-sends the prompt.
  - Copy Prompt button (clipboard icon with arrow): exports the full
    structured prompt (system context + user request + current code) to
    clipboard for pasting into an external LLM (ChatGPT, Claude web, etc.).
- Model info bar: shows current provider/model. Click to open Settings.
- Expand/Collapse button: toggle AI panel full-width or split view.
- Copy All: copies entire session log to clipboard.
- Clear Session: starts a fresh AI conversation.

PLAY MODE -- performance interface
- Shows the custom UI generated by the AI (if any), or auto-generated
  dial controls for all parameters.
- Preset strip: navigate presets (prev/next arrows), save with name,
  rename inline.
- AI prompt strip (when enabled): a compact input bar for describing
  changes without leaving Play mode. Supports voice input.
  Toggle this with the TogglePresetMode MIDI CC action.
- No code editor visible -- designed for mixing/performing.
- If the screen is black: no custom UI exists yet. Go to Build and ask
  the AI to "create a custom UI for this effect."

EXPLORE MODE -- community patches
- Browse: discover community patches, filter by category.
- Install: download a patch to your local presets.
- Publish: share your patch with the community.
- Vote: rate patches.
- Requires a free account (name only, set in Settings or Hub).

PRESETS
- Factory presets ship with each variant.
- User presets: save via the preset strip or Save Patch dialog.
  Each preset stores: DSP code, UI code, parameter values, parameter
  names/ranges, custom UI HTML/JS, metadata.
- Presets are .amorph files in ~/Library/Presets/Artists_in_DSP/AMORPH/.
- Snapshots: internal undo history (up to 30 states) within a session.
  Navigate with Next/Previous Snapshot.
- New Project: clears everything and starts fresh (with unsaved-changes
  warning).

PROVIDERS & MODELS
Set up in Settings > AI Provider. Each provider needs its own API key.

  Provider    | Models (best first)              | Notes
  ------------|----------------------------------|----------------------------
  OpenAI      | gpt-5.4, gpt-5.4-mini,          | Recommended starting point.
              | gpt-5.4-nano                     | Mini is fast and cheap.
  Anthropic   | claude-sonnet-4-6,               | Sonnet: fast, strong
              | claude-opus-4-7, claude-opus-4-6,| reasoning. Opus 4.7: best
              | claude-haiku-4-5                 | agentic. Haiku: cheap.
  Google      | gemini-3.5-flash,                | Stable, fast agentic model.
              | gemini-3.1-pro-preview,          | Pro: best quality (preview).
              | gemini-3.1-flash-lite            | Lite: very cheap.
  DeepSeek    | deepseek-v4-flash,               | Budget-friendly with good
              | deepseek-v4-pro                  | quality.

Voice input (mic button) ALWAYS requires an OpenAI key, regardless of
which provider is active for code generation.

SETTINGS REFERENCE
- AI Provider: select provider, enter/clear API key.
- Advanced (collapsible): model select, temperature, max tokens,
  agent turns.
  - Temperature: 0.00-1.00. Default 0.20. Lower = reliable, predictable
    DSP code. Raise to 0.40-0.60 for creative exploration (higher risk
    of compile errors).
  - Max tokens: completion output limit. 4096 default. Use 8192+ for
    complex multi-processor graphs or large synths. 16384+ for full UI
    rewrites.
  - Agent turns: max tool-call rounds per AI run. Default 20. The agent
    reads code, edits, compiles, checks errors, and iterates. Each
    read/edit/compile cycle is one turn. Lower for simple changes,
    raise for complex multi-step builds.
  - Auto-apply DSP: ON = AI changes go live immediately. OFF = you
    review a diff and click Apply or Discard. Keep OFF while learning.
- Appearance: Dark/Light theme.
- MIDI Control: assign MIDI CC numbers to actions:
  - Push-to-talk (voice input) -- default CC 119
  - Next/Previous preset
  - Next/Previous snapshot
  - Toggle Play / Build
  - Start/Stop agent
  - Apply/Discard agent draft
  - Toggle preset mode (switch Play between prompt and preset)
  Set any action to "Off" to disable.
- Developer (collapsible):
  - MCP Connection: connect external AI agents. Pick target
    (Claude Desktop / VS Code / Claude Code CLI), click Copy Config,
    paste into the agent's configuration.
  - Copy Agent Instructions: copy this guide to clipboard.
- About: version, variant, mode descriptions.
- Privacy: telemetry toggle (anonymous usage data only -- no code,
  prompts, or personal data), Reset Install ID, Privacy Policy link.

CONNECTING EXTERNAL AGENTS
Amorph runs a local HTTP JSON-RPC server on localhost (port 7331-7399).
- MCP-compatible agents (Claude Desktop, VS Code Copilot, Cursor,
  Windsurf, Continue.dev): use Settings > Developer > Copy Config.
  The bridge script translates MCP stdio to HTTP.
- CLI (for Codex, Aider, shell scripts, CI):
    python3 scripts/amorph_cli.py tools
    python3 scripts/amorph_cli.py call read_code '{"target":"dsp"}'
    python3 scripts/amorph_cli.py call get_error '{}'
    python3 scripts/amorph_cli.py --variant fx status
- Direct HTTP: POST JSON-RPC to http://localhost:<port>/mcp with
  Bearer token from the registry file.

PROMPT TIPS FOR GOOD RESULTS
- Be specific: "warm tape delay, 300ms, 60% feedback, high-frequency
  rolloff on repeats, subtle wow and flutter"
- Name the parameters you want: "with controls for delay time, feedback,
  wet/dry, and flutter amount"
- Reference known gear: "like a Roland Space Echo" or "Juno-60 chorus"
- For synths: specify oscillator, filter, envelope: "subtractive synth,
  two detuned saws, resonant 24dB low-pass, slow-attack ADSR"
- To iterate: "make the reverb darker" or "add a second delay tap at
  150ms panned left"
- For custom UIs: "create a visual UI with a large XY pad for cutoff
  and resonance, and knobs for the other parameters"

COMMON ISSUES
- No sound from Amorph FX: needs audio input -- check DAW routing.
- No sound from Amorph Instrument: needs MIDI input -- add a MIDI clip
  or connect a keyboard.
- Compile error: read the error message in the log. Ask the AI to fix
  it, or describe what went wrong.
- Black screen in Play: no custom UI generated. Go to Build, ask
  "create a custom UI for this effect."
- Voice input not working: requires OpenAI API key, even if using
  another provider for code generation.
- Settings dialog hidden: some DAWs put it behind the main window.
  Click the gear icon again or check behind the DAW.
